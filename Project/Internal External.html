Internal and External Style Sheets

When adding style to your website you can either include it in your html document or you can create a separate style sheet containing just your CSS.

Internal Style Sheets

With internal style sheets you create a style section in the head section of your html document. Then inside of this you write all of your CSS rules.

The image to the left shows the head section of a html document and how a style section has been created which contains a rule with a selector of p. As it has a generic selector this section would change the font of all paragraph sections on the webpage.





Here is a table to show the different advantages and disadvantages of internal style sheets.

Advantage Disadvantage
You can style pseudo-elements Style cannot be applied to multiple documents
Can create classes and use ids Slower loading times
File is complete, not reliant on external files Increased file size

External Style Sheets

With external style sheets you create a separate document containing your CSS and then link it to your html document.

This line must be included in the html document so that the CSS is applied to the html. Once this has been included then you create your classes. For both internal and external style sheets you call the classes in the same way uses the div command.

Here is a table to show the advantages and disadvantages of external style sheets.

Advantage Disadvantages
Cleaner structure of html file If CSS does not load or CSS file is lost, page will not be styled
Faster loading speed
Same classes can be used on multiple pages
As the page content is placed higher in the html file, a search engine is able to index your page much faster giving you a higher page ranking

Inline Style

Finally there is inline style. This is where you directly apply your style to the html section. Inside of an opening tag you include a style line to declare what CSS rules you want to apply to that element.

Here is a table to show the advantages and disadvantages of external style sheets.

Advantage Disadvantage
Easy to test and make quick fixes to You have to style every element and cannot reuse classes
Faster loading times Cannot style pseudo-elements
Smaller website Can accidentally override things
